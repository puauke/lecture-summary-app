### 方法1: コマンドプロンプトから起動
1. スタートメニューで「cmd」と検索して「コマンドプロンプト」を開く
2. 以下のコマンドを実行：

3. ブラウザが自動で開きます（開かない場合は http://localhost:8501 にアクセス）

### 方法2: PowerShellから起動
1. スタートメニューで「PowerShell」と検索
2. 以下のコマンドを実行：

## 🌐 インターネット経由で共有する方法（Streamlit Cloud）

### ✅ 既に設定済み
- リポジトリ: https://github.com/puauke/lecture-summary-app
- Streamlit Cloud: デプロイ済み

### 📤 共有URL取得方法
1. https://streamlit.io/cloud にログイン
2. あなたのアプリを選択
3. 表示されるURL（例: `https://lecture-summary-app-xxxxx.streamlit.app`）を友達に共有

### 🔑 重要：APIキーについて
- **各ユーザーが自分のAPIキーを入力する必要があります**
- あなたのAPIキーは共有されません（安全）
- 友達に以下を伝えてください：

```
1. Google AI Studio (https://ai.google.dev/) にアクセス
2. Googleアカウントでログイン
3. 「Get API Key」をクリック
4. 「Create API Key」で新しいキーを作成（無料）
5. アプリのサイドバーにそのキーを入力
```

## 📋 まだ設定していない項目

### 1. Streamlit Cloudの設定確認
- Secrets（秘密情報）の設定：不要（各ユーザーがAPIキーを入力）
- リソース制限：無料プランは512MB RAM
- カスタムドメイン：有料プラン限定

### 2. .envファイルの管理
- ⚠️ `.env` ファイルはGitHubにプッシュされません（.gitignoreで除外済み）
- ローカル開発用のみ
- 共有アプリでは使用されません

### 3. セキュリティ設定
- ✅ ファイルサイズ制限: 50MB
- ✅ 危険なURLフィルタリング
- ✅ SSRF攻撃対策
- ✅ 入力サニタイゼーション

## 🎯 アプリの特徴（共有用）

### ✅ マルチユーザー対応
- 各ユーザーが独自のAPIキーを使用
- データは各ユーザーのセッションで分離
- 他のユーザーのデータは見えない

### ✅ 無料で使える
- Google Gemini API: 無料枠で十分
- Streamlit Cloud: 無料プラン

### ✅ スマホ対応
- Streamlit Cloudのアプリはレスポンシブデザイン

## 🛠️ トラブルシューティング

### Streamlit Cloudでエラーが出る場合
1. GitHub リポジトリを確認（requirements.txt が最新か）
2. Streamlit Cloudのログを確認
3. アプリを「Reboot」する

### ローカルで起動できない場合
```powershell
# Streamlitが入っていない場合
pip install streamlit

# 依存関係を再インストール
pip install -r requirements.txt
```
